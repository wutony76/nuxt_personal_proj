# 功能規格書：六合彩信用盤（6hc-cd）

## 1. 文件目的與範圍

本規格定義 `/lottery/bg/6hc-cd` 信用盤頁面在本階段必備能力，重點是先完成玩法覆蓋與可切換展示。  
本次範圍不包含實際下注送單、賠率結算與後端對獎流程。

---

## 2. 必備玩法清單（來源：pcv2_0223 / app/config/6hc）

頁面必須提供以下 14 種玩法，且玩法 key 不得改名：

1. `banbo`
2. `duoxuanzhongyi`
3. `ixiaolian`
4. `lianma`
5. `qima`
6. `shengxiao`
7. `tema`

10. `wuxing`
11. `zhengma`
12. `zhengmate`

14. `zixuanbuzhong`

## 3. 頁面行為規範

### 3.1 玩法選單

- 頁面進入後需顯示全部玩法項目（14 個）。
- 每個項目需同時顯示：
  - 中文玩法名稱
  - 對應 key（供測試與對照）
- 點擊任一玩法後，該玩法需立即進入 active 狀態。
- 玩法切換需採 route 型式：
  - `/lottery/bg/6hc-cd/tema`
  - `/lottery/bg/6hc-cd/zhengma`
  - 其餘玩法以 `/lottery/bg/6hc-cd/<playKey>` 對應
- `/lottery/bg/6hc-cd` 需自動導向預設玩法路由（`tema`）。

### 3.2 內容區更新

- active 玩法切換時，內容區需同步更新：
  - 玩法名稱
  - 玩法 key
  - 來源模組（config_xxx.js）
  - 玩法說明
- 切換流程需有可觀測狀態：
  - loading
  - success
  - error（不可吞錯）
- 當玩法存在多個 `playTypeName`（例如 `tema` 的 `特码A`/`特码B`），頁面需提供子玩法切換按鈕，並同步更新目前子玩法。

### 3.3 狀態管理與分層

- 頁面狀態需以單一 `reactive` state object 為主。
- 事件與邏輯需區分：
  - `click`：UI 入口（使用者點擊玩法）
  - `actions`：流程控制（載入、狀態切換、錯誤處理）
  - `_handlers`：資料查找與私有轉換
- 玩法資料需集中在單一配置檔（`app/config/bg/6hc-cd.js`），頁面僅負責組裝與呈現。
- 頁面應拆分為至少兩個子元件：
  - `PlayTabs`：主玩法切換
  - `PlayPanel`：當前玩法詳情與子玩法切換
- `PlayPanel` 內的選項區需拆分為 14 個玩法專屬子元件（每個 key 一個），不得以單一通用元件覆蓋全部玩法：
  - `banbo`
  - `duoxuanzhongyi`
  - `ixiaolian`
  - `lianma`
  - `qima`
  - `shengxiao`
  - `tema`
  - `touweishu`
  - `weishulian`
  - `wuxing`
  - `zhengma`
  - `zhengmate`
  - `zhengterenzhong`
  - `zixuanbuzhong`

### 3.4 下注互動

- 內容區需提供可選投注項目（號碼或分組），並能多選。
- 使用者可輸入單注金額，且提供常用金額快速按鈕。
- 可用逗號輸入自訂投注項目並追加到已選清單。
- 送單時需至少包含：
  - 彩種識別（`LHC-CD`）
  - 當前玩法 key
  - 當前子玩法名稱
  - 已選投注項目（至少包含 `playId` 與顯示名稱）
  - 單注金額
- 本階段可先不傳 odds，後續由後端或下一階段補齊賠率欄位。
- 送單流程需有 loading/success/error，並回顯單號與拆單筆數。

### 3.5 參考 Layout（依提供截圖）

- 整體版面採三段式：
  - 左側：會員資訊卡（帳號、盤口類型、信用額度、剩餘額度）
  - 右側上方：開獎資訊列（彩種名、期號、6+1 顆球、倒計時）
  - 右側下方：玩法與下注操作區
- 右側玩法與下注區需依序包含以下區塊：
  1. 主玩法 Tabs（例如：`特码`、`正码`、`连码`、`五行`、`半波`、`生肖`...）
  2. 玩法資訊列（單注最低、單注最高、單期最高、抽水、佔成、兩面）
  3. 子玩法切換（例如 `特码A` / `特码B`）
  4. 當前玩法標題列（置中顯示玩法名，例如 `特码`）
  5. 下注選項表格區（至少含：`號碼`、`賠率`、`金額`）
  6. 底部操作列（快捷金額下拉、單注金額輸入、快速金額按鈕、重置、確定）
- 下注選項表格區需支援多欄並排（參考畫面為 4 組欄位重複），每筆投注項目至少具備：
  - 顯示名稱（號碼/選項）
  - 賠率顯示位
  - 金額輸入框
- 互動與視覺狀態需可辨識：
  - 已選投注項目需有 active 樣式
  - 不可下注項目需有 disabled 樣式
  - 當前 active 玩法與子玩法按鈕需有明確高亮

## 4. 非目標（本次不做）

- 不接入信用盤下注 API。
- 不實作完整賠率表格與金額輸入控制。
- 不搬移來源專案全量 UI 樣式。

---

## 5. 驗收條件

### 5.1 功能驗收

- 開啟 `/lottery/bg/6hc-cd` 可看到 14 個玩法按鈕。
- 逐一點擊 14 個玩法，內容區均可正確切換。
- 任一玩法無資料時，畫面能顯示明確錯誤訊息而非靜默失敗。
- 畫面區塊需符合 3.5 所述結構，至少包含主玩法 Tabs、子玩法切換、下注表格與底部快捷金額列。

### 5.2 文件驗收

- `openspec/specs/6hcCredit.md` 存在且包含完整玩法清單。
- 玩法 key 與 UI 顯示一致，不可出現遺漏或拼字不一致。
